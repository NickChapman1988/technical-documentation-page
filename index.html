Introduction

Grid and Flexbox are two fantastic layout tools to use when writing your CSS, but it can be hard to know which one to pick, or when to use one instead of the other. Both allow us to create complex layouts that were previously only possible by applying CSS hacks and/or JavaScript. Flexbox and CSS Grid share multiple similarities and many layouts can be solved with both.

Getting Started

Both Flexbox and CSS Grid create layouts using rows and columns, otherwise referred to as 'dimensions'. These dimensions operate along either an x-axis (rows) or y-axis (columns).

Flexbox is often referred to as 'one-dimensional' because it lays out items along either the horizontal or the vertical axis, so you have to decide whether you want a row-based or a column-based layout.

CSS Grid, on the other hand, is 'two-dimensional' as it allows you to work both horizontally and vertically, where you can precisely place grid items into cells defined by rows and columns.

However, this doesn't mean that you have to use Flexbox for all one-dimension layouts, and only use CSS Grid for two-dimensional layouts! It is possible to build 2D layouts with Flexbox, and CSS Grid will still work perfectly for many 1D layouts.

Using Flexbox

The main properties that Flexbox uses are display, flex-direction, justify-content and align-items.

Display and Flex-Direction Properties

To begin with, you'll need to create a container that includes the items you want to style.

  • The flexbox display property can be set to either flex or inline-flex; inline-flex is used to make the container inline, not the items inside the container.
  • The flex-direction has four different values it can be set to; row, column, row-reverse and column-reverse.
  • The row and column put child elements of the flex container into either a row or a column. The row-reverse and column-reverse values reverse the order of child items as they're listed in the HTML source code, so the last child would be placed first, the first child would be placed last and so forth.
Justify-Content Property

The justify-content property aligns items on the main axis, which is the same direction as the items in the container. It can be set to six different values: flex-start, flex-end, center, space-around, space-between and space-evenly.

  • flex-start aligns items in the container to the 'start' of the container - left for rows, or top for columns.
  • flex-end aligns items in the container to the 'end' of the container - right for rows, or bottom for columns.
  • center will align the items in the container to the center line of the container; this will be horizontally centered for rows, and vertically centered for columns
  • space-around adds an equal amount of space between each item and adds a "half-size" space at either end.
  • space-evenly is similar, but adds equal amounts of space between each item and at either end.
  • Finally, space-between takes all of the available space and shares it between items in the container evenly (with no spaces added at either end).
Align-Items Property

The align-items property helps position the items on the cross axis, which is perpendicular to the main axis of the items. This results in the opposite behaviour to justify-content! It can be set to four different values: stretch, flex-start, flex-end and center.

  • stretch is set by default, and will automatically stretch the items to the same height as the container (NOTE: this will be from top-to-bottom on row containers, and from left-to-right on column containers)
  • flex-start aligns items in the container to the 'start' of the container - left for columns, or top for rows.
  • flex-end aligns items in the container to the 'end' of the container - right for columns, or bottom for rows.
  • center will align the items in the container to the center line of the container; vertically for rows, horizontally for columns.

For more details and further usage examples, consult the Flexbox Documentation.

Using CSS Grid

The main properties used for CSS Grid are: display, grid-template-columns, grid-template-rows, grid-column, grid-row and gap.

Unlike Flexbox, CSS Grid uses these properties to style both the parent container and individual child elements.

Display and Grid-Template Properties
  • The CSS Grid display property can be set to grid or inline-grid in a similar way to Flexbox
  • grid-template-columns defines the line name and size of the grid-column property assigned to each item in a container.
  • grid-template-rows does the same thing, except it affects the grid-row property of each item in a container.

Both grid-template-columns and grid-template-rows have many different values that can be assigned to them to dictate the various size and arrangement of individual row and columns items. you can set the value for these properties to pixels (px), fractions (fr), character width (ch), auto, or none. You can mix and match the values.

For more details and further usage examples, consult the CSS Grid Documentation.

Grid-Row and Grid-Column Properties

The grid-column and grid-row properties both define an item's size and location within the grid container. Both can be set to a number, a name, or to a global CSS value such as inherit or unset.

The Gap Property

The CSS Grid gap property sets the space between columns and rows inside of a container. Its value can be pixels(px), font-size(em), percentages, a calc() function, and more. You can also specify if it is for rows only: row-gap or for columns only: column-gap.

Flexbox vs. CSS Grid

So which should you be using: Flexbox or CSS Grid? The answer is both!

The reality is that CSS Grid can do things that Flexbox can't do, and Flexbox can do things that CSS Grid can't do, but they can work together - a grid item can be a flexbox container, and a flex item can be a grid container.

Layouts need to be assessed individually on a case by case basis to determine whether to use Flexbox, CSS Grid, or a mix of both. It really depends on what the layout needs to look like, and what content is present.

CSS Grid focuses on precise content placement; each item is a grid cell, lined up along both a horizontal and a vertical axis.

Properties such as grid-template-rows and grid-template-columns and utilities like the fraction unit (fr) let you precisely calculate everything, making CSS Grid especially suitable for creating unusual layouts like broken, asymmetrical and overlapping layouts.

CSS Grid also makes it possible to create responsive layouts without using media queries, by making use of auto layout, minmax(), repeat() and auto-fill.

Generally, the strengths of CSS Grid lend themselves to creating structured, ordered and precisely-positioned layouts and will best suit any content that needs those qualities - whole page layouts, literal grids,

Flexbox focuses on content flow rather than content placement. Widths (or heights) of flex items are determined by the content of the item. Flex items grow and shrink according to their inner content and the available space.

In short, Flexbox enables you to allocate space and align items flexibly. It also allows you to decide how your content should behave when there's too much or not enough space on the screen. Using the flex-grow and flex-shrink properties, you can achieve a completely fluid layout that optimises the allocation of flex items at every viewpoint size.

Flexbox generally works best with content that needs to be flexible, intrinsicly-sized and responsive; things like centering items, adjusting the last item(s) of lists, sticking footers to the bottom of pages, and creating responsive menus. There are even more Flexbox usage examples in the MDN Docs.

The ultimate answer to the Flexbox vs. CSS Grid question is that both have their pros and cons, and both are better suited to some things than the other. Don't stick to one or the other, but always think through which serves your goals better for any given layout. If you need to, use both!

To learn more about these two fantastic tools, there are plenty of resources available. MDN Web Docs is a great resource for not only CSS but also for HTML, JavaScript, APIs and much more. W3Schools is also a great resource for all things web-development-related. Most of all, play around with both Flexbox and CSS Grid and see what they can do for you.
